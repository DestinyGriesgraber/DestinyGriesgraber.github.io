Well Done

There are two ways to make sure javascript cannot be read by the layman from the developers side. Based on my research, obfuscation is a popular method of abstracting the source code so as to not be legible by the regular human. Aother method is to disable right clicking and the ability to view source code/console. The other method is to only have javascript that records and sends data to the server, have all the bits you want secret on the server, have it calculate and send back the result to the browser. The main drawbacks of obfuscation is that the code can be cracked. Either by apps or the exact script used to obfuscate it reversed. Disabling developer tools and right click will only make it a little bit harder for the code to be seen. The equivalent of having a motion sensing light to keep out a burglar. For the server side processing, obviously it means you need to have more resources in the server to render out your functions and send them back to the browser.
